## はじめに

**AI Story Builder**は、AIと協働して小説を創作するための統合執筆環境です。「面倒な作業はAIに、創造性はあなたに」をコンセプトに、物語の種からプロット構築、執筆、校正までを一貫してサポートします。

👉 **[📖 漫画でわかるAiStoryBuilder はこちら](漫画でわかるAiStoryBuilder.md)**

### 主な特徴

- **マルチモーダル創作**: 画像や音声からインスピレーションを得て物語を生成
- **強力な執筆支援**: 続きを書く、描写強化、壁打ちチャットなど多彩なAIツール
- **統合データ管理**: 用語集、相関図、年表、伏線管理をサイドバーに集約
- **プライバシー重視**: データはローカルに保存され、オフライン（ローカルLLM）でも動作可能

### Windows / Linux

1. [GitHubのリリースページ](https://github.com/bookendless/aistorybuilder/releases)からインストーラーをダウンロードします。
   - Windows: `.exe` または `.msi`
   - Linux: `.AppImage` または `.deb`

1. リリースページから `.apk` ファイルをダウンロードし、端末にインストールしてください。

## アプリの概要

### 画面構成

アプリは大きく3つのエリアで構成されています：

1. **左サイドバー (ナビゲーション)**: 制作ステップ（物語の種、キャラクター、プロット...）を切り替えます。
2. **中央エリア (メインワークスペース)**: 各ステップの編集作業を行います。
3. **右サイドバー (ツール & 支援)**: AI支援パネルや、用語集・相関図などのツールにアクセスします。

### 制作ステップ

1. **物語の種**: テーマ、舞台設定、ターゲット読者などを定義
2. **キャラクター**: 登場人物のプロフィールを作成（AI自動生成対応）
3. **構成**: 起承転結や三幕構成などのフレームワークで構造を作成
4. **あらすじ**: 物語全体の流れをテキスト化
5. **章立て**: 各章の概要と構成を設計
6. **執筆**: 本文の執筆（AI入力支援、リライト機能あり）
7. **分析**: 作品全体のバランスや整合性をAIが分析・評価
8. **エクスポート**: 完成した作品をテキスト、Markdown、HTML等で出力

## 初期設定 (AI設定)

AI機能を使用するには、まずAIプロバイダーの設定が必要です。画面右上の「設定」アイコンから「AI設定」を開いてください。

### 対応プロバイダー

1. **Anthropic Claude**
   - **Claude Fable 5 / Sonnet 5 / Opus 4.8 / Haiku 4.5**: 自然で文学的な日本語表現が得意。Fable 5は最上位のMythosクラス、Sonnet 5はOpus級の品質をSonnet価格で提供。いずれも最大100万トークンの長文に強く、整合性チェックにも優れます。（Opus 4.7 / 4.6 / 4.5 / Sonnet 4.6 / 4.5 も選択可能）

2. **OpenAI**
   - **GPT-5.6（Sol・Terra・Luna）/ 5.5 / 5.4（mini・nano）/ o3・o4-mini**: 高度な推論能力とマルチモーダル対応。論理チェックや画像分析に。GPT-5.6/5.5/5.4は最大105万トークンに対応します。

3. **xAI Grok**
   - **Grok 4.5 / 4.3 / Grok 4.20（Reasoning・Non-Reasoning）**: Grok 4.5はxAI最高知能の最新モデル（最大50万トークン）。Grok 4.3/4.20は長文（最大100万トークン）に強く、長編の文脈維持に最適です。

4. **Google Gemini**
   - **Gemini 3.5 Flash / 3.1 Pro / 3 Flash / 2.5 Pro**: 超長文コンテキスト（最大200万トークン）とマルチモーダル処理に強み。高速・高知能のGA版3.5 Flashが主力。高度な推論には3.1 Pro (Preview)も選択できます。

5. **ローカルLLM (Ollama / LM Studio)**
   - **完全オフライン**で動作可能。プライバシーを最優先する場合に選択してください。
   - 設定例（Ollama）: エンドポイントに `http://localhost:11434` を入力
   - **最大プロンプト長**: 送信するプロンプトの上限文字数を設定できます（既定: 12000文字）。長文の推敲・改善が途中で切れる場合はお使いのモデルのコンテキスト長に合わせて値を大きくしてください。低スペック環境で不安定な場合は小さくします。上限を超えたプロンプトは切り詰められ、その際は画面に警告が表示されます。

## 制作ワークフロー

### 1. 物語の種 (Idea)

「画像から物語を作る」「音声から物語を作る」機能を使えば、視覚・聴覚情報から初期設定を一気に生成できます。手動でテーマや「物語の引き」を入力することも可能です。

#### AIおまかせ骨組み生成

新しいプロジェクトを作成する際、「プロジェクトの説明」を入力すると、その下に **「AIおまかせ骨組み生成」** トグルが現れます。オンにして作成すると、説明文を**物語の種**として、AIが次の3つをまとめて下書きします。

- プロット基本設定（メインテーマ・舞台設定・フック要素・主人公の目標・主要な障害・物語の結末）
- 主要キャラクター 2〜3人のドラフト
- 推奨される構成テンプレート

生成はプロジェクト作成後に**バックグラウンド**で進み（左下のインジケータで確認・キャンセル可能）、完了すると確認モーダルで内容をプレビューしてから**「反映する」「破棄する」**を選べます。説明文が空のとき、またはトグルがオフのときは従来どおりの作成フローになります。

### 2. キャラクター (Characters)

基本情報を入力して「AIで強化」ボタンを押すと、詳細な性格や背景をAIが提案します。

- **憑依チャット**: 作成したキャラクターになりきったAIと会話できます。口調の確認や深掘りに便利です。

### 3. プロット (Plot)

「起承転結」「ヒーローズ・ジャーニー」「ビートシート」などのテンプレートを選択し、AIにプロット案を出力させることができます。

### 4. 執筆 (Draft)

Markdown対応のエディタで本文を執筆します。

- **禅 (Zen) モード**: 集中するために余計なUIを隠します。
- **AI支援コマンド**: テキストを選択して、リライト、描写強化、続きの生成などを指示できます。

### 5. 管理ツール

右サイドバーの「ツール」タブには以下の機能があります。これらは全てのステップから参照可能です。

- **用語集**: 固有名詞管理
- **相関図**: キャラクター関係図
- **タイムライン**: 年表
- **伏線トラッカー**: 伏線の回収状況管理
- **感情マップ**: 感情推移グラフ（※既定では非表示。表示するには下記の設定が必要です）

> **表示するツールの切り替え**: 「ツール」タブの「プロジェクトツール」見出し横にある歯車アイコン（⚙）から、各ツールの表示/非表示を切り替えられます。**感情マップ**は初期状態ではオフのため、利用したい場合はここでオンにしてください。

## AI生成の管理（実行状況・完了通知・反映確認）

AI生成は時間がかかることがあります。本アプリでは、生成中でも他のステップで作業を続けられます。

### 実行中インジケータ
生成中は画面**左下**に「AI生成中 n件」のインジケータが常に表示されます。クリックすると実行中タスクの一覧（内容・経過時間・進捗）が開き、**どのステップからでも個別にキャンセル**できます。ステップを移動しても生成は止まりません（戻れば進捗を確認できます）。

### 完了通知（トースト）
生成が完了すると、**どのステップにいても**画面右上にトーストで通知されます。全章一括生成のような長時間の処理は、見逃さないよう**手動で閉じるまで残る**通知になります。

### 反映確認モーダル（重い生成）
作品の構造を大きく変える**重い生成**（プロット構成・章立て・あらすじ・全体あらすじ・キャラクター生成）は、完了しても**すぐには反映されません**。完了トーストの「**確認する**」ボタンを押すと、生成内容のプレビューが開き、**「反映する」「破棄する」**を選べます。

- 別のステップに移動している間に完了した場合でも、完了トーストの「確認する」からいつでも反映/破棄できます。
- 基本設定の一括提案・文体調整など軽い生成は、従来どおりすぐに反映されます。
- キャラクター提案やプロット一貫性チェックは、従来どおり専用モーダルで内容を選択・適用します。

### 創造ポイント（あらすじ・章立て）

**あらすじ／章立て**の生成では、AIは推奨案で最後まで生成しつつ、**「作者の判断によって物語が分かれる箇所」** を2〜4点、確認モーダル内にカードとして提示します。各カードには現在の推奨案と、1〜3個の**別案（選んだ場合の帰結つき）**が並びます。

- 別案ボタンを押すと、その方向で当該生成を**やり直し**ます（出力全体が新しい案に置き換わり、再度確認モーダルが開きます）。
- 気にならなければ無視してそのまま「反映する」で構いません（**ブロッキングしません**）。
- この機能は **AI設定 → 「創造ポイントを提案する」** でオン/オフできます（既定はオン）。

### 先回り生成（次のステップを裏で用意）

あるステップを完了して次へ進むと、AIが**次のステップを先回りして裏で生成**しておきます。次の3つの遷移が対象です。

- **プロット構成**を完了 → **あらすじ**を先回り生成
- **あらすじ**を完了 → **章立て**を先回り生成
- **章立て**を完了 → **草案**（最初のまだ書いていない章ひとつ）を先回り生成

生成は左下のインジケータで確認・キャンセルでき、完了すると確認モーダルで内容をプレビューしてから**「反映する」「破棄する」**を選べます（先回り中に内容を変更して再度離脱した場合は、古い結果を破棄して作り直します）。

- この機能は **AI設定 → 「次のステップを先回り生成する」** でオン/オフできます（**既定はオフ**）。
- ⚠ オンにすると、ステップを進めるたびに**自動でAPI課金が発生**します。費用にご注意ください。

## 続編構成

完成済みプロジェクト（全ステップ進捗100%）を元に、AIが続編の土台を自動生成する機能です。ホーム画面の「続編構成」ボタンから起動します。

### 使い方

続編構成はモーダル内で完結する5ステップのウィザードです。

#### ステップ1：元プロジェクトの選択

完成済みプロジェクトの一覧から、続編の元にしたい作品を選択します。

- **詳細抽出モード**: チェックを入れると、各章の「あらすじ」に加えて「本文（草稿）」も要約の材料として使用します。より高精度な分析が得られますが、AI呼び出し回数が増えるため時間とAPIコストが増加します。

#### ステップ2：前作の要素抽出（AI処理）

AIが前作の全章を自動分析します。以下の項目が抽出されます。

| 抽出項目 | 内容 |
| -------- | ---- |
| 全体要約 | 前作ストーリーのダイジェスト |
| キャラクターの成長・変化 | 各人物が辿った変化の軌跡 |
| 関係性の変化 | 登場人物間の関係の変遷 |
| 世界観の変化 | 前作を通じて変わった世界の状態 |
| 未解決の要素 | 読者の期待を引き継ぐ伏線・フック |

処理中はキャンセルボタンで中断できます。中断した場合、次回モーダルを開いた際に再開を選択できます。

#### ステップ3：抽出結果の確認・編集

AIが生成した抽出結果を自由に編集できます。この内容が続編生成のインプットになるため、気になる点があれば修正してから次に進んでください。

#### ステップ4：続編要素の確認・編集

抽出結果を元にAIが生成した続編の要素を確認・編集します。

- **あらすじ**: 続編全体のストーリーの流れ
- **プロット項目**: テーマ、舞台・状況設定、フック、主人公の目標、主要な障害
- **引き継ぎ情報**: キャラクター（成長を反映した状態）と世界観設定（前作からの変化メモ付き）が自動で引き継がれます

#### ステップ5：プロジェクト作成

続編プロジェクトのタイトルを入力して「続編プロジェクトを作成」を押します。

- 未入力の場合は「元タイトル（続編）」が自動設定されます
- 作成後はキャラクターステップが開き、引き継いだキャラクターの確認・追加から作業を続けられます
- 章立て・執筆以降は通常の制作フローと同じです

### 注意事項

- **対象プロジェクト**: 全ステップの進捗が100%のプロジェクトのみが対象です
- **マルチ世代対応**: 続編をさらに続編にすることも可能です（続編番号が自動インクリメントされます）
- **生成範囲**: あらすじ・プロットまでを自動生成します。章立て以降はユーザーが通常フローで制作してください

## 小説の取り込み（インポート）

手持ちの小説（断片・未完成作品）のプレーンな本文を読み込み、AIが分析して AiStoryBuilder の各項目（キャラクター・プロット・あらすじなど）に再構成し、新規プロジェクトとして取り込む機能です。ホーム画面の「小説を取り込む」ボタンから起動します。

> **重要**: 取り込んだ原文はそのまま保存され、AIによる書き換えは行いません（「再構成」であって「リライト」ではありません）。

### 使い方

モーダル内で完結する4ステップのウィザードです。

#### ステップ1：本文の入力

- **ファイル追加**: `.txt` / `.md` のテキストファイルを複数選択できます。複数ファイルは一覧で読み込み順を並べ替えでき、上から順に連結されます。
- **直接貼り付け**: テキストエリアに本文を直接貼り付けることもできます。
- 入力すると、合計文字数とおおよそのAI呼び出し回数の目安が表示されます（分量が多いほど時間とAPIコストが増加します）。

#### ステップ2：AI解析（AI処理）

AIが本文を分析します。長い本文は自動的に分割して処理されます（章マーカーがなくても動作します）。

- **概要系**（あらすじ・テーマ・プロット基本・ジャンル）は、本文を要約・集約してから抽出します。
- **登場人物**は本文全体を走査し、脇役も含めて抽出したうえで、同一人物を自動的に名寄せ・統合します。愛称・呼び方違い（例：山田太郎／太郎／タロちゃん）も、AIが抽出した別名と仕上げの同一人物判定で1人にまとめられます（判定に迷う場合は誤統合を避けるため別人のまま残ります。確認画面で修正してください）。
- **文体**は、機械計測（会話比率・文長・人称・文末の硬軟）とAI分析（基本文体・比喩・感情描写・トーン）を組み合わせて推測し、アプリの**文体設定**の選択肢から最も近いものを自動選択します。判定材料が足りない項目は無理に埋めず「未設定」のままになります。

処理中はキャンセルボタンで中断できます。中断後も、次回モーダルを開いた際に確認画面から再開できます。

#### ステップ3：解析結果の確認・編集

AIが推定したタイトル案・ジャンル・あらすじ・プロット基本項目・登場人物の一覧を確認・編集します。

> AIの推定には、原文に無い創作や取りこぼしが含まれる場合があります。**必ず内容を確認・修正してください。** 人物は追加・削除も可能です。

- **文体設定（原文から推測）**: 原文の文体に近い設定（基本文体・人称・硬軟・リズム・比喩表現・会話比率・感情描写・トーン）が自動選択され、AIが読み取った文体の特徴メモも表示されます。プロジェクト作成後のAI執筆（続きを書く・リバイズ等）はこの設定に沿って生成されるため、原文に近い雰囲気を保ちやすくなります。各項目はドロップダウンで変更でき、作成後もプロジェクト設定からいつでも調整できます。
- **文体見本（原文からの抜粋）**: 原文の中間部から自動抽出した逐語抜粋が「文体見本」として登録されます。AI執筆（続きを書く・章全体の生成・リバイズ）時に「この文章の雰囲気・文体に合わせる」見本としてそのままAIに渡されるため、選択肢ベースの文体設定よりも直接的に原文の文体を再現できます。作品の最も「らしい」一節に差し替えるとさらに効果的です（空にすると使用されません）。作成後は、ドラフトステップのアシスタントパネル「文体設定」→「見本を編集」からいつでも変更・再抽出できます。

#### ステップ4：プロジェクト作成

タイトルを入力して「プロジェクトを作成」を押すと、新規プロジェクトが作成され、キャラクターステップが開きます。

- 取り込んだ原文は「インポート本文」という章に保存されます。章立てステップの**「本文を章に分割」**ツール（後述）で複数の章に分割できます。

### 注意事項

- **対象形式**: v1 では `.txt` / `.md` とテキストの直接貼り付けに対応します（`.docx` 等は非対応）。文字コードは UTF-8 / Shift-JIS / UTF-16 を自動判定し、改行コード（CRLF・CRのみ・LF）も自動的に正規化されます。
- **投入先**: 解析結果は常に新規プロジェクトとして作成されます（既存プロジェクトへのマージは行いません）。
- **自動投入される項目**: タイトル・ジャンル・想定読者・あらすじ・プロット基本・登場人物・文体設定・原文ドラフトです。章分割は章立てステップの「本文を章に分割」で、プロット構成（起承転結等）はプロット構成詳細の「AIで構成を推定」または通常フローで、取り込み後に作成してください。

### 本文を章に分割する（章立てステップ）

取り込んだ「インポート本文」など、本文（ドラフト）を持つ章は、章立てステップの章一覧にあるハサミアイコン「**本文を章に分割**」から複数の章に分割できます。

- **自動検出**: `第3章` `第三話` のほか、`一話　○○` `０１章` `02章` のような「第」のない見出し、`プロローグ` `序章` `エピローグ` `幕間` などの特殊見出し、`# 見出し` 形式の markdown 見出しを自動検出して分割位置の候補にします。
- **区切り記号の扱い**: `◇◆◇` や `***` などの記号は「場面転換」であって章の区切りではないため、既定では分割位置にしません。章として扱いたい場合はトグル「区切り記号も候補に含める」をオンにします。
- **手動調整**: 各章カードでタイトルの編集・分割の解除（前の章へ統合）ができます。「分割位置を追加」では本文の行を検索して任意の行頭を新しい章の開始位置に指定できるため、見出しのない作品も手動で分割できます。
- **AI提案（任意）**: 「AIに分割位置を提案させる」ボタンで、AIに章の区切り候補を提案させられます。提案は分割位置として表示されるだけで、**「分割を適用」を押すまで保存されません**。通常の分割はAIを一切使いません。
- **本文は逐語のまま**: 分割は本文の切り出しのみで、AIによる書き換え・要約は行いません。分割後の各章の本文を連結すると元の本文と完全に一致します（画面下部に「合計◯字 / 原文◯字」を常時表示）。

### プロット構成のAI推定（プロット構成詳細）

プロット構成詳細（PlotStep2）の構成スタイル選択の横にある「**AIで構成を推定**」ボタンで、あらすじと章一覧から、6種類の構成（起承転結・三幕構成・四幕構成・ヒーローズ・ジャーニー・ビートシート・ミステリー・サスペンス）のどれに当てはまるかをAIが判定し、各段階の内容を下書きします。

- 推定結果は確認画面に表示され、**「この構成を適用」を押すまで反映されません**。
- 作品情報から読み取れない段階は空欄のまま提示されます（無理に埋めません）。適用後に自由に編集できます。
- アシスタントパネルの「構成生成」（選択中の構成を創作的に展開する機能）とは別物です。取り込んだ既存作品の整理には「構成を推定」、新規の展開づくりには「構成生成」が向いています。

### うまくいかないとき（推奨：本文の直接貼り付け）

分析結果が本文と食い違う・破綻する、文字化けする、エラーになる、一部プロバイダー（例：Gemini）でセンシティブ判定される——といった場合は、**ファイルではなく本文を入力欄に直接貼り付けて**お試しください。

- 直接貼り付けはクリップボードの正しい文字コードでテキストが渡るため、**ファイル固有の文字化け**（Shift-JIS 等の取り込み）や、**一部プロバイダーの入力フィルタ誤検知**を回避できることがあります。
- 取り込み後の「インポート本文」が文字化けしている場合も、本文を貼り付け直して取り込み直すと改善します（※既に作成済みのプロジェクトの本文は遡って修正されません）。

## トラブルシューティング

### AIが応答しない

- APIキーが正しいか確認してください。
- ネットワーク接続を確認してください。
- ローカルLLMの場合、サーバー（Ollama等）がバックグラウンドで起動しているか確認してください。

### プロジェクトデータについて

データはブラウザ（アプリ）内のデータベースに自動保存されます。重要なデータは定期的に「エクスポート」からJSON形式でバックアップを取ることをお勧めします。
